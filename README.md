Задание 1

1. Создание проекта

Создайте новый React проект при помощи Vite.

2. Установка зависимостей

Установите все необходимые зависимости в проект (npm i @reduxjs/toolkit react-redux redux).

3. Структура приложения

Удалите все лишние файлы из проекта.

В src создайте папку redux для Redux файлов и папку components для будущих компонентов.

4. Проверка

Запустите приложение, проверьте, что всё работает.
Задание 2

Создать slice для управления состоянием списка задач (todos) с использованием Redux Toolkit.

1. Импортируйте необходимые модули:

Откройте файл, в котором вы будете создавать ваш slice.
Импортируйте функцию `createSlice` из `@reduxjs/toolkit`.

2. Определите начальное состояние:

Создайте переменную для начального состояния. Начальное состояние будет пустым массивом, так как у нас изначально нет задач.

3. Создайте slice:

Используйте функцию `createSlice`, чтобы создать новый slice. Определите следующие параметры:

- `name`: Имя slice. Назовите его, например, `todos`.
- `initialState`: Установите начальное состояние, которое вы определили ранее.
- `reducers`: Определите объект с функциями-редьюсерами для управления состоянием.

4. Добавьте редьюсер для добавления новой задачи:

Создайте функцию-редьюсер с именем `addTodo`.
Функция должна принимать текущее состояние и действие (action).
Внутри функции используйте метод `push`, чтобы добавить новую задачу из `action.payload` в текущее состояние (массив задач).

5. Добавьте редьюсер для переключения статуса выполнения задачи:

Создайте функцию-редьюсер с именем `toggleTodo`.
Функция должна принимать текущее состояние и действие (action).
Внутри функции найдите задачу по `id`, переданному в `action.payload`.
Переключите значение свойства `completed` найденной задачи (если задача выполнена, сделайте её невыполненной, и наоборот).

6. Добавьте редьюсер для удаления задачи:

Создайте функцию-редьюсер с именем `deleteTodo`.
Функция должна принимать текущее состояние и действие (action).
Внутри функции используйте метод `filter`, чтобы вернуть новый массив без задачи с `id`, переданным в `action.payload`.

7. Экспортируйте действия:

После создания slice, экспортируйте действия, которые автоматически создает Redux Toolkit. Экспортируйте `addTodo`, `toggleTodo` и `deleteTodo`.

8. Экспортируйте редьюсер:

Экспортируйте редьюсер slice, который будет использоваться в хранилище Redux.
Задание 3

2. Импортируйте функцию configureStore:

Откройте файл `store.js`.
Импортируйте функцию `configureStore` из библиотеки `@reduxjs/toolkit`.

3. Импортируйте редьюсер задач:

Импортируйте редьюсер задач, который вы создали ранее, `todosReducer` из файла `todosSlice.js`.

4. Создайте хранилище с использованием configureStore:

Вызовите функцию `configureStore` и передайте объект конфигурации.

- В объекте конфигурации укажите ключ `reducer`.
- Внутри ключа `reducer` создайте объект, который будет содержать редьюсер вашего приложения.

5. Экспортируйте созданное хранилище:

Экспортируйте хранилище, чтобы его можно было использовать в других частях приложения.

Передайте хранилище в качестве свойства `store` компонента `Provider`.

Задание 5

Импортируйте компонент `TodoItem`, который будет использоваться для отображения каждой задачи.

2. Создание функционального компонента `TodoList`:

Определите `TodoList` как функциональный компонент.
Импортируйте хук `useSelector` для получения списка задач из состояния Redux.

- Этот хук принимает функцию, которая получает все состояния и возвращает часть состояния, которая нас интересует.
- В данном случае, мы возвращаем массив задач из состояния.

3. Возврат JSX:

Возвращайте JSX, который описывает структуру нашего компонента.

- Создайте HTML-элемент `div` с заголовком `Todo List`.
- Внутри элемента `ul` используйте метод `map` для итерации по массиву задач и создания компонента `TodoItem` для каждой задачи.
- Передавайте каждую задачу в качестве пропса `todo` и используем её `id` в качестве уникального ключа.

4. Экспорт компонента:

Задание 6

Создание компонента TodoForm

1. Создание файла компонента

Внутри вашей папки компонентов (например, `src/components`) создайте новый файл с именем `TodoForm.js`.

2. Импорт необходимых модулей

Импортируйте React и хук `useState` из `react`.
Импортируйте хук `useDispatch` из `react-redux`.
Импортируйте действие `addTodo` из вашего файла `todosSlice.js`.

3. Создание функционального компонента TodoForm

Создайте функциональный компонент `TodoForm`.
Используйте хук `useState` для управления локальным состоянием поля ввода текста новой задачи.
Используйте хук `useDispatch` для создания функции, которая будет отправлять действия в хранилище Redux.

4. Создание формы

Создайте HTML-форму, которая будет содержать:

- Поле ввода для текста новой задачи.
- Кнопку для добавления новой задачи.

Привяжите значение поля ввода к состоянию `text`.
Добавьте обработчик события `onChange` к полю ввода, чтобы обновлять состояние `text` при изменении текста.
Добавьте обработчик события `onSubmit` к форме для обработки отправки формы.

5. Реализация логики добавления задачи

В обработчике `handleSubmit` предотвратите стандартное поведение формы.
Проверьте, что текст задачи не пустой (удалите лишние пробелы).
Отправьте действие `addTodo` с новой задачей в хранилище Redux. Задача должна содержать уникальный `id`, текст и статус выполнения (по умолчанию `false`).
Очистите поле ввода после добавления задачи.

6. Экспорт компонента

Экспортируйте компонент `TodoForm`, чтобы его можно было использовать в других частях вашего приложения.
